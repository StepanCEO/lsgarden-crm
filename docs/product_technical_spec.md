# PlantFlow CRM - Продуктовая и техническая спецификация

Дата: 2026-06-21

## 1. Продуктовая часть

### 1.1 Цель продукта
CRM должна автоматизировать полный цикл работы магазина растений:
- входящее обращение -> квалификация -> продажа -> доставка/самовывоз -> повторная коммуникация
- контроль работы сотрудников по скриптам и дедлайнам
- сквозной учет клиентов, переписок, заказов, остатков и аналитики

### 1.2 Пользователи
- Администратор
- Фронт
- Бек
- Гибрид
- Контент
- Локомотив

### 1.3 Основные рабочие сценарии
1. Сотрудник входит в систему, видит дашборд и очередь задач.
2. Новое входящее обращение приходит в единое окно.
3. CRM связывает обращение с клиентом или создает новую карточку.
4. Менеджер использует скрипт ответа, проверяет остатки и отправляет сообщение.
5. Система ставит follow-up, если клиент не ответил.
6. Склад получает данные из 1С, CRM обновляет остатки и статус товара.
7. Администратор управляет ролями, словарями, импортом, логами и аналитикой.

### 1.4 Функциональные блоки

- Авторазлогин после 5 часов бездействия
- Блокировка прав при аномальной активности
- Логирование всех действий

## 2. Техническая часть

### 2.1 Целевой стек
- Backend: Django
- DB: PostgreSQL
- Frontend: Django templates + static assets
- Deploy: Docker / Docker Compose
- Background jobs: Celery или аналог
- Cache / broker: Redis, если нужны фоновые задачи и очереди

### 2.2 Архитектура
- Django project `config`
- app `crm`
- отдельные приложения можно выделить позже:
  - `accounts`
  - `integrations`
  - `warehouse`
  - `knowledge`
  - `analytics`
  - `audit`

### 2.3 Модель данных
Минимальный набор сущностей:
- User / EmployeeProfile
- Client
- Message
- Task
- Product
- KnowledgeArticle
- NewsItem
- AuditEntry

Расширения для production:
- RolePermission
- ChannelAccount
- IntegrationEvent
- ImportJob
- ExportJob
- ClientMergeLog
- BankStatementRow
- OneCRecord
- FollowUpRule
- ScriptRule
- TestAttempt

### 2.4 Интеграции
- Telegram Bot API
- VK API
- WhatsApp provider API
- SMTP / IMAP для почты
- Flowwow API
- Avito email/parsing или scraping
- 1C обмен
- bank CSV import
- S3-compatible storage для медиа

### 2.5 Безопасность
- CSRF
- session security
- password policies
- audit trail
- role-based access control
- rate limiting
- secret management via env vars
- logging and alerting

### 2.6 Хранение и деплой
- PostgreSQL as primary DB
- media outside app server
- Dockerized deployment
- Nginx reverse proxy
- Gunicorn for Django WSGI
- HTTPS
- nightly backups
- restore verification

### 2.7 Нефункциональные требования
- response time acceptable for main screens
- stable under multiple users
- immutable audit log
- recoverable from backup
- easy to extend with integrations
- clear admin workflows

### 2.8 Приемочные критерии
Считаем продукт готовым, если:
- все роли входят и видят только свои разделы
- есть клиенты, переписки, тикеты, товары, аналитика
- работает импорт 1С и банка
- работает склейка дублей
- работают скрипты и follow-up
- есть логирование действий
- есть бэкап-план
- есть инструкция запуска и деплоя
